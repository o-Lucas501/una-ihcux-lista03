# una-ihcux-lista03
Heurística 1 de Nielsen: Visibilidade do Status do Sistema

R — Resumo
A heurística de Visibilidade do Status do Sistema estabelece que o sistema deve sempre manter o usuário informado sobre o que está acontecendo, por meio de feedback apropriado e em tempo razoável.

A — Aplicação

Exibir indicadores de carregamento (spinners, barras de progresso)
Mostrar confirmações de ações (mensagens de sucesso/erro)
Atualizar estados em tempo real quando possível
Informar atrasos ou processos longos com estimativas

D — Diretrizes

Feedback deve ser claro, imediato e relevante
Evitar silêncio do sistema após ações do usuário
Usar linguagem simples e compreensível
Priorizar consistência nos sinais de status

M — Métricas

Tempo médio de resposta percebido pelo usuário
Taxa de abandono em fluxos com carregamento
Número de erros por falta de feedback
Satisfação do usuário (UX surveys)

E — Exemplos

Barra de progresso em uploads/downloads
Mensagem “Salvo com sucesso” após envio de formulário
Indicador “Digitando...” em chats
Skeleton screens durante carregamento de páginas
